# 인사관리 앱 분리 출시 플랜

## 개요

**StoreBase (슈퍼앱)** 코드 복사 → **네비게이션만 HR용으로 단순화** → 배포

같은 Supabase DB 공유, 같은 인증 시스템 사용

---

## 문제될 부분

| 항목 | 문제 여부 | 비고 |
|------|----------|------|
| DB 연동 | ✅ 없음 | 같은 Supabase 공유 |
| 인증 | ✅ 없음 | 같은 Auth 공유 |
| UI 컴포넌트 | ✅ 없음 | 그대로 복사 |
| HR 기능 코드 | ✅ 없음 | 그대로 복사 |
| **네비게이션** | ⚡ 수정 필요 | HR 기능만 보이게 단순화 |
| 코드 동기화 | 🔸 관리 필요 | StoreBase 변경 시 HR앱에 복사 |

---

## HR 앱에 포함될 기능 (5개 모듈)

| 모듈 | 기능 |
|------|------|
| attendance | 출퇴근, 근태 관리 |
| employee_setting | 직원 관리, 급여 설정 |
| time_table_manage | 시간표/스케줄 관리 |
| store_shift | 매장 근무 설정 |
| delegate_role | 권한/역할 관리 |

---

## 작업 목록

1. 새 Flutter 프로젝트 생성
2. StoreBase 코드 전체 복사
3. 네비게이션/홈화면을 HR 전용으로 수정 (HR 5개 기능만 표시)
4. 앱 아이콘/스플래시/브랜딩 변경
5. 앱스토어 등록

---

## 장점

- 데이터 한 곳에 저장 → HR앱 사용자가 나중에 StoreBase로 업그레이드해도 데이터 유지
- 한 번 가입으로 양쪽 앱 사용 가능
- StoreBase 변경사항을 HR앱에 그대로 적용 가능

---

## 결론

**기술적 문제 없음.** 네비게이션만 단순화하면 끝.
